# 🚀 Сравнение производительности: PostgreSQL против Redis

Это приложение загружает тестовую базу данных **world** через Liquibase, а затем выполняет один и тот же запрос дважды: сначала к PostgreSQL, затем к Redis (где данные предварительно кешируются).

В консоль выводится время выполнения каждого запроса, что позволяет наглядно сравнить производительность.

---

## 🧰 Технологии

- Java 21
- Maven
- Liquibase (миграции)
- PostgreSQL 17
- Redis 7
- Docker / Docker Compose

---

## 📦 Что делает приложение?

1. Заселяет базу данных **world** (таблицы `city`, `country`, `country_language`) — миграции находятся в каталоге `changelog/`.
2. Выполняет запрос к PostgreSQL и замеряет время выполнения.
3. Сохраняет результат в Redis (кеширование).
4. Выполняет запрос к Redis и замеряет время получения тех же данных.
5. Выводит в консоль время выполнения обоих запросов для сравнения.

---

## 🐳 Запуск через Docker Compose (рекомендуемый способ)

### 1. Соберите JAR приложения локально

```bash
mvn clean package
```
Убедитесь, что в каталоге `target/` появился файл:

```text
final4-1.0-SNAPSHOT.jar
```

или аналогичный.

### 2. Запустите все сервисы одной командой

```bash
docker-compose up -d
```

Команда поднимет:

- PostgreSQL (порт `5432`)
- Redis (порт `6379`)
- Ваше приложение (однократный запуск миграций и тестового запроса)

После завершения работы контейнер приложения остановится (`restart: "no"`), а PostgreSQL и Redis продолжат работу.

### 3. Посмотрите результат

```bash
docker-compose logs app
```

Пример вывода:

```text
Redis:       35 ms
PostgreSQL:  40 ms
```

### 4. Остановка и очистка

```bash
docker-compose down -v
```

Команда остановит все контейнеры и удалит тома с данными.

## 🖥️ Локальный запуск (без Docker)

### Требования

- Java 21
- PostgreSQL 17 (запущен локально)
- Redis 7 (запущен локально)

### Настройка

Создайте базу данных `world` и пользователя:

- Логин: `postgres`
- Пароль: `postgres`

Либо измените параметры подключения в коде или переменных окружения.

### Переменные окружения

| Переменная | Значение по умолчанию | Описание |
|------------|----------------------|----------|
| `DB_URL` | `jdbc:postgresql://localhost:5432/world` | URL подключения к PostgreSQL |
| `DB_USER` | `postgres` | Пользователь PostgreSQL |
| `DB_PASSWORD` | `postgres` | Пароль PostgreSQL |
| `REDIS_HOST` | `localhost` | Хост Redis |
| `REDIS_PORT` | `6379` | Порт Redis |

### Запуск

```bash
java -jar target/final4-1.0-SNAPSHOT.jar
```

## 📁 Структура миграций (Liquibase)

```text
changelog/
├── v1.0/
│   ├── 001-create-tables.yml
│   ├── 002-insert-city.yml
│   ├── 003-insert-country.yml
│   └── 004-insert-country-language.yml
└── changelog-master.xml
```

### Описание миграций

| Файл | Назначение |
|-------|------------|
| `001-create-tables.yml` | Создание таблиц `city`, `country`, `country_language` |
| `002-insert-city.yml` | Загрузка данных о городах |
| `003-insert-country.yml` | Загрузка данных о странах |
| `004-insert-country-language.yml` | Загрузка данных о языках |
| `changelog-master.xml` | Главный файл Liquibase, подключающий все изменения |

---

## 🔄 Применение миграций

Все миграции Liquibase применяются автоматически при первом запуске приложения.

После успешного выполнения база данных будет заполнена тестовыми данными, и приложение сразу выполнит сравнение производительности PostgreSQL и Redis.
